## 6) P4：UI/UX 品質提升（設計一致性 + 緊湊型管理風格）

> **設計方向：** 整體走「管理系統（management）」風格，佈局緊湊、資訊密度高，避免過多留白。

### 6.1 高優先（用戶體驗直接影響）

- [ ] **空狀態標準化**：建立統一 `<EmptyState>` 元件（含圖示 + 一致的文字樣式），取代各頁零散的純文字提示。
- [ ] **資料載入狀態**：目前只有表單送出有 loading 指示；資料表、列表的資料撈取期間應有 skeleton 或 spinner 提示。
- [ ] **行內欄位驗證錯誤**：錯誤訊息目前只出現在頁面頂端 `StatusBanner`；常見錯誤（如密碼不符）應在欄位旁即時顯示。
- [ ] **手機版側欄**：小螢幕目前無法收合/展開側欄，需加入漢堡選單或抽屜式 nav。

### 6.2 中優先（設計一致性）

- [ ] **設計 Token 統一**：CSS 變數已定義（`globals.css`），但元件仍大量使用硬編 Tailwind class（`bg-amber-600`、`text-amber-700`）。需建立統一使用規則，未來換色只改一處。
- [ ] **按鈕樣式規範**：主要操作（實心 `bg-amber-600`）vs 表格內圖示按鈕（外框 `border-emerald-300`）無明確規則。應制定填色 vs 外框的使用時機。
- [ ] **頁面標題區塊**：部分頁面有 amber 色麵包屑標籤在 `h2` 上方，部分沒有，需統一格式。
- [ ] **表格緊湊化**：配合管理風格，統一調整 row height、padding、字體大小，使各頁資料表視覺一致且密度較高。
- [ ] **手機版資料表**：目前僅橫向捲動，寬表（如追蹤矩陣）在手機上難以操作；考慮卡片式列呈現。

### 6.3 低優先（開發維護品質）

- [ ] **登入頁圖示大小**：角色選擇圖示（132×132px）在手機上佔用過多垂直空間，應依螢幕尺寸縮小。
- [ ] **DataManagementTable 共用元件**：`group-management-table`、`class-management-table`、`member-management-table` 邏輯高度重複，可抽成共用 `<DataManagementTable>` 降低維護成本。
- [ ] **大型 Panel 拆分**：`tracking-management-panel.tsx`（49KB）與 `group-study-management-panel.tsx`（54KB）過於龐大，應拆成更小的子元件。

### 6.4 驗證與驗收

- [ ] Developer 驗證：`npm run lint`、`npm run check:encoding`、`npm run build`
- [ ] UAT 驗證：
  - [ ] 各頁面資料表視覺密度一致，符合管理系統風格
  - [ ] 空狀態、載入狀態出現時機正確且風格統一
  - [ ] 手機版側欄可正常收合/展開
  - [ ] 按鈕樣式語意清楚（操作類 vs 危險類 vs 次要類）
