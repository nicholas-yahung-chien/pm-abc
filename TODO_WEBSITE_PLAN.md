## 1) 報表內容分析摘要（保留）

### 1.1 工作表與用途
- `通訊錄`：學員/教練聯絡資訊。
- `R&R`：小組角色與分工。
- `小組團隊章程`：共識、規範、目標、互動方式。
- `讀書會`：每週時間、地點、形式、內容。
- `導讀分配`：導讀範圍、頁次、負責學員。
- `追蹤表`：學習進度追蹤（核心）。

### 1.2 追蹤表關鍵規則
- 階層：`大項 > 小項 > 細項`。
- 值型別：`bool / number / date / enum / text`。
- 可附加外部連結與附件。
- 需保留逾期提醒與完成率統計能力。

## 2) 新需求整理（登入 + 權限 + 操作）

### 2.1 使用者類型
- 管理員（admin）
- 教練（coach）
- 學員（member）

### 2.2 登入與註冊流程
- 首頁新增登入入口：`教練登入`、`學員登入`（另提供管理員入口）。
- 教練：`唯一 email + 密碼` 註冊，狀態先 `pending`，需管理員審核後才可使用。
- 管理員：唯一審核者；預設帳密 `root / root`（第一版先依需求實作）。
- 教練註冊申請時：寄送審核通知信到管理員登記信箱。
- 學員：不自行註冊，由教練建立（唯一 email）；學員以 email 密碼免登入（OTP/Magic Link）。

### 2.3 班別/小組與歸屬規則
- 教練可建立班別，並管理班別的課程時間與大綱。
- 教練可在班別下建立小組，且可指派任一教練為小組擁有者。
- 教練可建立學員並指派到班別/小組。
- 教練登入後可選擇其被指派班別/小組進入組內管理。
- 學員登入後可選擇其被指派班別/小組進入組內操作。
- 管理員可使用 admin/coach/member 三種身分能力。

### 2.4 小組內模組與行為規則
- 每個小組都有：`追蹤表 / 通訊錄 / 團隊章程 / R&R / 讀書會 / 導讀分配`。
- 教練與學員可查看其他班別/小組內容，但僅唯讀。
- 進入小組預設頁：`追蹤表`。
- 追蹤表：
  - 教練可建立與管理追蹤項目。
  - 學員只能回報是否完成（勾選）。
- 通訊錄：
  - 教練資訊固定顯示最上方，且由教練帳號資料維護。
  - 教練與學員都可編輯通訊錄中的學員資訊。
- 團隊章程：教練與學員皆可編輯。
- R&R：教練與學員皆可建立角色、管理角色、指派角色。
- 讀書會：教練與學員皆可建立與管理時間/地點/形式/內容。
- 導讀分配：教練與學員皆可建立項目、頁次範圍、指派學員。

## 3) 權限模型與系統規劃（調整版）

### 3.1 RBAC + Scope（角色 + 範圍）
- `admin`：全域可管理，含教練審核、教練帳號 CRUD、班別/小組/成員全管理。
- `coach`：
  - 其所屬班別/小組可管理。
  - 非所屬班別/小組唯讀。
- `member`：
  - 其所屬班別/小組可協作（依模組規則）。
  - 追蹤表僅能更新自己的完成回報欄位。
  - 非所屬班別/小組唯讀。

### 3.2 權限矩陣（核心）

| 功能 | Admin | Coach(所屬) | Member(所屬) | 非所屬 |
|---|---|---|---|---|
| 教練審核/教練帳號管理 | 可 | 不可 | 不可 | 不可 |
| 班別建立/管理 | 可 | 可 | 不可 | 唯讀 |
| 小組建立/管理/教練指派 | 可 | 可 | 不可 | 唯讀 |
| 學員建立/指派 | 可 | 可 | 不可 | 唯讀 |
| 追蹤項目建立/編輯/排序 | 可 | 可 | 不可 | 唯讀 |
| 追蹤回報（完成勾選） | 可 | 可 | 可（本人） | 唯讀 |
| 通訊錄學員資訊編輯 | 可 | 可 | 可 | 唯讀 |
| 團隊章程編輯 | 可 | 可 | 可 | 唯讀 |
| R&R 建立/管理/指派 | 可 | 可 | 可 | 唯讀 |
| 讀書會建立/管理 | 可 | 可 | 可 | 唯讀 |
| 導讀分配建立/管理 | 可 | 可 | 可 | 唯讀 |

### 3.3 登入與導頁規劃
- `/login`：角色入口頁（教練登入 / 學員登入 / 管理員登入）。
- `/login/coach`：email + password。
- `/login/member`：email -> OTP/Magic Link。
- `/login/admin`：username/password（第一版採 `root/root`）。
- 登入成功後進 `班別/小組選擇器`，再進入小組。
- 小組首頁預設導向：`/groups/:groupId/tracking`。

## 4) 資料模型調整（Supabase）

### 4.1 新增/調整資料表
- `user_profiles`
  - `id`（對應 auth user）
  - `email`（unique）
  - `display_name`
  - `primary_role`（admin/coach/member）
  - `coach_approval_status`（pending/approved/rejected）
- `coach_applications`
  - `email`（unique）
  - `password_hash`（或暫存註冊憑證）
  - `status`（pending/approved/rejected）
  - `reviewed_by`, `reviewed_at`
- `admin_settings`
  - `notification_email`（接收教練審核通知）
- `class_memberships` / `group_memberships`
  - 使用者與班別/小組關聯（含 role in scope）
- `group_coach_owners`
  - 小組擁有教練（可多教練，至少一位 owner）
- `tracking_items`
  - 追蹤項目結構與排序
- `tracking_member_reports`
  - 學員進度回報（限定本人可改）

### 4.2 RLS 原則
- Admin：全資料可讀寫。
- Coach：僅可寫入其所屬 scope；其他 scope 僅 SELECT。
- Member：僅可寫入授權欄位（例如本人 tracking 回報）；其他 scope 僅 SELECT。
